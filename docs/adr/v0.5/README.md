# v0.5 architecture decision records

These records track the decisions listed in
[`V0.5-DESIGN.md` §14](../../V0.5-DESIGN.md#14-decisions-required-before-public-alpha).
An ADR records a proposed or accepted design; it does not imply that the
corresponding runtime behavior has shipped. An ADR closes its corresponding
design decision only after acceptance under its lifecycle rule. Status is
merge-derived; a candidate does not accept itself, and this index does not
copy a mutable lifecycle value into immutable candidate bytes.
